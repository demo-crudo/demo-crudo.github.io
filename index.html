Crudo · TOML-defined commerce

A secure store backend, written in TOML.

Crudo turns one declarative store.toml file into a functional digital-goods backend — catalog, accounts, sessions, a balance ledger, and payment flows served over HTTP. This page is a static client: every product, price, and account detail here is live data from the configured API.

Browse the catalog Account dashboard
  • Declared, not coded

    Endpoints, SQL actions, authentication rules, and request limits are defined in configuration — the behavior of this store comes from config/store.toml.

  • Security built in

    Bearer-token sessions, hashed credentials, ALTCHA proof-of-work on sign-in and registration, and per-endpoint rate limiting.

  • HTTP 402 payments

    Purchases draw on the account balance; when funds are short the API answers with a payment-required response and per-user deposit destinations.
